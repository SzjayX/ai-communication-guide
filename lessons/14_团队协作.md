# 课程 14：团队协作 — 把 AI 能力推广到整个团队

## 学习目标
- 理解团队 AI 推广失败的根本原因，避免重蹈覆辙
- 掌握"三层分级"培训体系，针对不同水平设计不同的成长路径
- 学会三阶段推广法：启蒙 → 能力建设 → 融入工作流
- 能设计团队的 AI 使用规范、模板库和安全政策
- 掌握处理团队中 AI 抵触、滥用和退潮的策略
- 从"团队里最会用 AI 的人"变成"让整个团队都会用 AI 的人"

---

## 核心概念

### 1. 一个人的效率 × 10 ≠ 团队效率 × 10

先看一个真实的模式：

```
你：经过 3 个月的学习和实践，AI 已经帮你省了 40% 的时间。
   你写的方案质量更高了，代码审查更快了，周报 5 分钟搞定。

你的同事：还在手动做一切。一个方案写 3 天，周报写 1 小时。
   他们知道你在用 AI，但觉得"那是你厉害，我学不会"。

团队整体效率：只提升了你一个人的 40%。其他人的效率没有任何变化。
```

**这就是本课要解决的核心问题**：你不是要把 13 课的内容灌给每个人，而是建立一个**系统**，让团队成员用最小的学习成本，获得最大的 AI 使用能力。

---

### 2. 为什么团队 AI 推广经常失败？

在讲"怎么做"之前，必须理解"为什么难"。大多数团队 AI 推广失败不是因为 AI 不好用，而是因为推广方式本身有问题。

#### 失败的六种典型模式

```
┌──────────────────────────────────────────────────────────┐
│            团队 AI 推广 · 六种失败模式                       │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ❌ 模式 1：一次培训然后放羊                               │
│     "周五下午搞个 AI 培训，然后就靠大家自学了"               │
│     → 培训后第一周 30% 的人会用，一个月后只剩 5%            │
│                                                          │
│  ❌ 模式 2：丢一堆资料让大家看                              │
│     "我把学习资料发到群里了，大家有空看看"                   │
│     → 没人看。连你都不会看                                │
│                                                          │
│  ❌ 模式 3：强制要求但没给工具                              │
│     "从下个月起，所有技术方案必须先用 AI 审查一遍"           │
│     → 阳奉阴违。大家敷衍了事，变成走形式                    │
│                                                          │
│  ❌ 模式 4：只推工具不推方法                               │
│     "我们买了 ChatGPT Team 版，大家用起来！"               │
│     → 工具有了，但每个人都在瞎用。有人问"今天天气怎么样"     │
│                                                          │
│  ❌ 模式 5：只靠一个人推动                                 │
│     "小王是咱们团队的 AI 专家，大家有问题找他"              │
│     → 小王变成了免费客服，自己的活干不完，最后也放弃了       │
│                                                          │
│  ❌ 模式 6：没有和实际工作流结合                            │
│     "AI 很重要，大家要重视"                                │
│     → 但日常工作的流程、模板、评审标准没有任何变化           │
│     → 用 AI 变成"额外的事"，而不是"工作的方式"             │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

#### 失败的根本原因：三个断层

| 断层 | 表现 | 后果 |
|------|------|------|
| **认知断层** | 推广者觉得"这很简单"，学习者觉得"这太难了" | 教学内容不匹配受众水平 |
| **激励断层** | 用 AI 没有看得见的好处，不用 AI 没有后果 | 缺乏行为改变的动力 |
| **系统断层** | 工作流程不嵌入 AI，用了 AI 也要手动复制粘贴 | 使用 AI 是额外负担而非效率工具 |

### 4. 三层培训体系详解

#### 🟢 新手层：2 小时启蒙工作坊

**目标**：从"不敢用、不会用"到"敢用、能用对一次"。

**关键认知**：对新手的培训，不要追求"全面"，要追求"立刻能用"。2 小时后，他们能拿着一个模板完成自己工作中的第一个 AI 任务——这就是成功。

**工作坊设计**：

```
┌──────────────────────────────────────────────────────────┐
│          新手工作坊 · 2 小时议程                            │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  0:00-0:15  破冰：你用过 AI 吗？（收集恐惧和期待）          │
│             "大家说说，你们觉得 AI 能帮你们做什么？          │
│              有什么担心的？"                               │
│                                                          │
│  0:15-0:35  震撼演示（最关键的部分）                        │
│             选 2 个他们日常工作中的真实痛点，现场演示：       │
│             - 痛点 1：写一份竞品分析（产品团队）             │
│             - 痛点 2：写一封客户邮件（销售团队）             │
│             对比：模糊指令 vs 结构化指令的输出质量差异       │
│             "看到了吗？不是 AI 不行，是说没说清楚"          │
│                                                          │
│  0:35-0:55  核心方法：RTBSF 简化版                         │
│             只讲 5 个字母，每个字母配一个他们行业的例子：     │
│             R：你要 AI 扮演谁？                            │
│             T：你要 AI 做什么？                            │
│             B：背景是什么？（这最重要！）                    │
│             S：分几步做？                                  │
│             F：输出什么格式？                              │
│                                                          │
│  0:55-1:15  动手实操                                       │
│             每人用自己的一个真实任务，按 RTBSF 写一条指令     │
│             现场发给 AI，看结果，讨论                        │
│                                                          │
│  1:15-1:35  避坑三原则                                      │
│             1. AI 说的不一定对——重要信息要核实              │
│             2. 不要输入机密信息                            │
│             3. 第一版不满意很正常——改了再问                  │
│                                                          │
│  1:35-1:50  发"一张纸指南" + 模板                          │
│             每人领一份可以贴在显示器旁边的"一张纸指南"       │
│             每人领 3 个自己岗位最常用的指令模板              │
│                                                          │
│  1:50-2:00  Q&A + "这周你用 AI 完成一个任务"的承诺           │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**"一张纸指南"模板**：

这是一个可以打印出来贴在显示器旁边的精简指南：

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃            🤖 和 AI 高效协作 · 一张纸指南                   ┃
┣━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┫
┃                                                          ┃
┃  📝 写指令 = 角色 + 任务 + 背景 + 步骤 + 格式               ┃
┃                                                          ┃
┃  ✅ 好的：                                                ┃
┃  "你是资深产品经理。帮我校验这份 PRD 的逻辑完整性。          ┃
┃   产品是面向开发者的 API 工具，受众是有 3 年+ 经验的工程师。  ┃
┃   请按'用户场景→功能→验收标准'逐项检查。                    ┃
┃   输出：问题列表 + 严重度 + 修改建议，表格形式。"             ┃
┃                                                          ┃
┃  ❌ 不好的：                                              ┃
┃  "帮我看看这个 PRD 怎么样"                                 ┃
┃                                                          ┃
┃  🔄 第一版不满意？说清楚哪里不对，再问一次                   ┃
┃                                                          ┃
┃  ⚠️ 关键信息要核实 | 不要输入机密 | 给 AI 例子它更懂你       ┃
┃                                                          ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
```

#### 🟡 进阶层：4 小时工作坊 + 2 周实践

**目标**：从"能用对一次"到"能稳定写出高质量指令"。

**进阶层不需要再讲基础**。重点放在三个提升维度：

```
进阶层三大模块（4 小时）：

模块 1：示例驱动（1 小时）
  → 从课程 07 中提取：正例和反例怎么用
  → 实操：每个人给自己常用的场景写一组正例+反例

模块 2：迭代式沟通 + 常见错误（1.5 小时）
  → 从课程 08+11 中提取：五步迭代法 + 10 个常见错误
  → 实操：拿一个"不好的指令"，一步步改成"好的指令"
  → 经典练习：每个人拿出自己过去"翻车"的一次 AI 对话，
    分析犯了哪个错误，当场重写指令

模块 3：模板实战（1.5 小时）
  → 从课程 09 中提取：把好指令固化为模板
  → 实操：每个人为自己最常用的 3 个场景建模板
  → 团队活动：各组交换模板，互相测试，收集反馈
```

**2 周实践计划**：

```
第 1 周：
  □ 每天至少用 AI 完成 1 个工作任务
  □ 每次记录：用了什么指令？结果如何？改进空间？
  □ 周五：组内分享 1 个"本周最佳指令"

第 2 周：
  □ 把 3 个常用指令做成模板
  □ 用模板完成同样任务，对比效率和质量
  □ 周五：提交 3 个模板到团队模板库
```

#### 🔴 专家层：完整学习路径

**目标**：能独立设计 Agent、维护模板库、带新人。

专家层的人不需要培训，需要的是**资源和责任**：

```
专家的成长路径：

📚 学习资源：
  □ 完成全部 15 课（这门课本身）
  □ 每月深度研究 1 个 AI 新能力
  □ 参与外部 AI 社区，带回前沿实践

🔧 实践责任：
  □ 维护团队的模板库（审核、更新、清理）
  □ 每月做 1 次"AI 最佳实践"团队分享
  □ 帮助新人写出第一条高质量指令（结对指导）

🏆 成长指标：
  □ 能独立设计一个 5 工具的 Agent
  □ 能帮助 5 个以上同事显著提升 AI 使用能力
  □ 能为团队引入 1 个新的 AI 使用场景
```

**专家层人数不需要多**：一个 20 人的团队，有 2-3 个专家就足够了。他们的角色是"放大器"——帮助其他人用得好。

### 5. 三阶段实施详解

#### 阶段 1：启蒙（第 1-2 周）

**核心目标**：让至少 50% 的人完成第一次"成功的 AI 使用体验"。

**关键动作**：

```
第 1 周：
  □ 做全员工作坊（2 小时）
  □ 发"一张纸指南"（纸质版 + 电子版）
  □ 领导发一条公开消息：分享自己用 AI 的真实案例
  □ 创建 #ai-sharing 频道（Slack/飞书/钉钉）

第 2 周：
  □ 每个人承诺"这周用 AI 完成 1 个真实任务"
  □ 在频道里分享（匿名也可以）：
    "我用了 AI 做 ____，省了 ____ 分钟，效果 ____"
  □ 周五：快速回顾——收集困惑，解答高频问题
```

**领导示范（最重要的一步）**：

如果领导自己不公开使用 AI，推广成功率减半。示范不是发一条"大家要多用 AI"，而是具体展示：

```
❌ 糟糕的示范：
"AI 是未来趋势，大家要积极学习使用。"

✅ 好的示范：
"这周我用 AI 帮我们分析了 3 个竞品的最新动态。
我的做法：先收集了竞品的公开资料，然后用 RTBSF 框架写了一条指令，
让 AI 从'功能变化、定价变化、用户反馈变化'三个维度分析。
最终输出了一份 1500 字的分析报告——过去这需要我花一个下午，
现在只用了 40 分钟。下面是完整的指令，大家可以直接拿去改改用。"
```

**区别**：前者是口号，后者是可复制的行动指南。

#### 阶段 2：能力建设（第 3-6 周）

**核心目标**：让每个岗位的人掌握自己领域的高频 AI 用法。

**2.1 分角色专场培训**

不同岗位的 AI 用法差异很大。不要混在一起培训——产品经理不想听怎么写代码，工程师不想听怎么写营销文案。

```
🔧 技术团队专场（工程师）
  核心技能：
  ├── 用 AI 写代码和调试（带思维链）
  ├── 用 AI 审查代码（带安全/性能/可读性检查清单）
  ├── 用 AI 写技术文档和 API 文档
  └── 用 AI 评估技术方案（带多方案对比）
  关键提醒：AI 写的代码必须 Review，不能直接上线

📊 产品团队专场（PM）
  核心技能：
  ├── 用 AI 做竞品分析（带对比维度）
  ├── 用 AI 写 PRD 和用户故事
  ├── 用 AI 分析用户反馈（提取高频关键词和情感倾向）
  └── 用 AI 做产品规划和优先级排序
  关键提醒：AI 不了解你的用户，你才是用户需求的最终判断者

📣 运营/市场专场
  核心技能：
  ├── 用 AI 策划内容选题（带热点和用户兴趣分析）
  ├── 用 AI 写营销文案（带受众画像和风格约束）
  ├── 用 AI 优化 SEO（关键词分析和标题测试）
  └── 用 AI 做社媒内容分发策略
  关键提醒：AI 不懂你的品牌调性，给它足够的品牌指南

👔 管理/HR/行政专场
  核心技能：
  ├── 用 AI 写汇报和总结
  ├── 用 AI 起草制度和流程文档
  ├── 用 AI 做会议纪要和行动项提取
  └── 用 AI 辅助招聘（JD 优化、面试问题设计）
  关键提醒：涉及人事决策的，AI 只给参考，决策必须由人做
```

**每个专场 2 小时**：1 小时演示 + 0.5 小时实操 + 0.5 小时 Q&A。

**2.2 建立团队模板库**

模板库是团队 AI 能力的"基础设施"。建得好，所有人受益；建不好，没人会用。

**模板库的设计原则**：

```
1. 按角色组织，不是按技术主题
   ✅   product/竞品分析.md
       dev/代码审查.md
       marketing/营销文案.md
   ❌  prompt-engineering/chain-of-thought.md
       （只有专家层的人能看懂这种分类）

2. 每个模板包含完整的"使用说明"
   不只是指令本身，还有：什么场景用、参数怎么填、
   常见问题怎么调、谁维护的

3. 标注"新人友好度"
   🟢 新人可用（填 2-3 个参数即可）
   🟡 需要一定经验（需要理解 5+ 个参数的含义）
   🔴 专家级（需要理解 Agent 概念）

4. 版本控制
   用 Git 管理，每次改模板都要写"改了什么、为什么改"
```

**推荐的模板库目录结构**：

```
📁 team-ai-templates/
│
├── 📁 product/           ← 产品团队
│   ├── 🟢 竞品分析.md
│   ├── 🟢 PRD 大纲生成.md
│   ├── 🟡 用户反馈分析.md
│   ├── 🟡 产品规划.md
│   └── 🔴 需求评审 Agent.md
│
├── 📁 dev/               ← 技术团队
│   ├── 🟢 代码审查.md
│   ├── 🟢 Bug 报告分析.md
│   ├── 🟡 技术方案评估.md
│   ├── 🟡 API 文档生成.md
│   └── 🔴 架构设计 Agent.md
│
├── 📁 marketing/         ← 运营/市场
│   ├── 🟢 营销邮件.md
│   ├── 🟢 社媒文案.md
│   ├── 🟡 内容选题策划.md
│   └── 🟡 活动方案策划.md
│
```

#### 阶段 3：融入工作流（第 7 周起）

**核心目标**：让 AI 从"偶尔用的工具"变成"工作方式的一部分"。

**3.1 把 AI 嵌入现有流程**

不要让"用 AI"成为额外的一步。要把它嵌入已有的工作流：

```
嵌入点示例：

代码审查流程：
  原来：写代码 → 提 PR → 同事 Review → 修改
  嵌入：写代码 → AI 预审（用审查模板）→ 提 PR → 同事 Review
  → 同事的 Review 负担减少 50%

产品需求评审：
  原来：写 PRD → 发给同事 → 开会讨论
  嵌入：写 PRD → AI 逻辑检查（用 PRD 校验模板）
  → 修正明显问题 → 发给同事 → 开会讨论
  → 评审会上不再讨论"格式不对""逻辑跳跃"这类低级问题

周报/月报：
  原来：手动写，每次 30-60 分钟
  嵌入：每天花 1 分钟记录关键事件到模板 →
  周五 AI 自动生成周报初稿 → 5 分钟修改 → 发送
```

**3.2 设计正向激励**

不要惩罚不用 AI 的人。要奖励用得好的人：

```
激励方式（由轻到重）：

🥇 公开表扬
   "本周最佳 AI 实践：XX 用 AI 帮我们发现了线上 Bug 的根因，
   从日志分析到根因定位只用了 15 分钟。这是他的方法：[链接]"

🥈 物质激励
   每月评选"AI 应用之星"，奖励 200 元购书卡 / 一次午餐

🥉 职业发展
   将"AI 使用能力"纳入晋升评估的一个参考维度
   （注意：是参考维度，不是硬指标——避免形式主义）

🏆 团队层面的认可
   季度"最佳模板"评选，获奖模板的创建者获得额外奖励
```

**3.3 建立轻量化的评估体系**

不需要复杂的 KPI。用"脉搏调查"（Pulse Check）就够了：

```
每月一次，3 个问题（各 1-5 分）：

Q1：过去一个月，你用 AI 完成了多少工作任务？
    1 = 几乎没用  5 = 每天都用

Q2：AI 对你工作质量的提升程度？
    1 = 没什么提升  5 = 提升非常显著

Q3：你在使用 AI 时遇到的最大障碍是什么？
    （开放式文本）

追踪趋势比追求绝对值重要。
如果 Q1 从 2.1 → 3.5，说明推广在见效。
如果 Q3 反复出现"不知道怎么写指令"，说明需要加强培训。
```

### 6. 处理团队中的"AI 抵触"

推广中最棘手的不是"不会用"，而是"不想用"。抵触通常来自以下五类原因：

| 抵触类型 | 典型言论 | 背后的真实担忧 | 应对方式 |
|---------|---------|--------------|---------|
| **恐惧型** | "AI 会取代我的工作" | 职业不安全感 | 展示 AI 是工具不是替代品；强调"会用 AI 的人"比"不会用的人"更有竞争力 |
| **不屑型** | "AI 写的东西不行，还不如我自己写" | 对自己专业能力的骄傲 | 让他用一个具体任务 PK：他 vs 他用 AI。赢了说明他确实强，输了说明 AI 有用——都不亏 |
| **忙碌型** | "没时间学，手头活都干不完" | 学习成本焦虑 | 给他 1 个最简单的模板，让他下次直接套用。30 秒填参数 → 见证效果 → 自然愿意学更多 |
| **安全型** | "公司数据不能放到 AI 里去" | 合规和安全担忧 | 制定清晰的 AI 使用安全政策，标明"可以/不可以"的边界。不确定性是恐惧的来源 |
| **惯性型** | "我一直这样做也没问题啊" | 对改变的本能抗拒 | 不打嘴炮，用对比说话。同一个任务，A 用老方法做，B 用 AI 辅助做。对比时间和质量 |

**通用原则**：不要试图用道理说服情绪。面对抵触，先认可担忧（"你的担心是对的，确实有风险"），再展示如何管理风险（"所以我们要这样做……"）。

### 8. 阶段性检查清单

| 阶段 | 时间 | 成功指标 |
|------|------|---------|
| **启蒙** | 第 2 周末 | ≥50% 的人完成了第一次"成功的 AI 使用"并在频道分享 |

## 关键要点总结

1. **推广失败的原因是可预测的**：六种失败模式（一次放羊、丢资料、强制无工具、推工具不推方法、靠一人、不嵌入流程）几乎覆盖了所有失败案例
2. **三层分级，因材施教**：新手（2 小时启蒙）→ 进阶（4 小时 + 实践）→ 专家（完整学习 + 带人），不要一刀切
3. **三阶段推进，循序渐进**：启蒙（让人敢用）→ 能力建设（让人会用）→ 融入工作流（让人习惯用）
4. **领导示范是最强杠杆**：领导公开分享自己的 AI 使用案例，胜过 10 场培训
5. **模板库是基础设施**：按角色组织、标注难度、配使用说明、版本控制
6. **安全政策消除恐惧**：一页纸说清楚什么能输入什么不能，不确定性是抵触的最大来源
7. **不要惩罚不用的，奖励用得好的**：正向激励比强制要求有效 10 倍

## 实践活动

### 练习 1：评估你的团队现状（15 分钟）

如果你的团队还没有系统化使用 AI，请回答：

### 练习 2：设计"一张纸指南"（20 分钟）

为你所在的团队设计一份"一张纸指南"：

1. 用本课的模板作为起点
2. 把 RTBSF 示例换成你自己岗位的真实案例
3. 加上你团队最常用的 2-3 个场景的快速指令模板
4. 控制在 A4 一页以内

### 练习 3：写一份团队 AI 推广计划（30 分钟）

按本课的三阶段框架，为你的团队写一份推广计划：

1. **阶段 1（第 1-2 周）**：工作坊怎么安排？谁来主讲？用什么案例？
2. **阶段 2（第 3-6 周）**：按岗位分几个专场？每个专场的核心内容？
3. **阶段 3（第 7 周起）**：AI 嵌入哪些现有流程？怎么激励？

### 练习 4：建立第一个团队模板（20 分钟）

选一个你团队中最常见的任务，建一个"新人友好"的模板：
- 🟢 难度标记
- 清晰的参数（不超过 5 个）
- 使用说明（什么场景用、怎么填参数）
- 一个正例和一个反例
