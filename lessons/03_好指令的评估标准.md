# 课程 03：好指令的评估标准（什么样的叫好用）

## 学习目标
- 掌握评价指令质量的标准
- 学会自我评估和改进指令
- 理解可重复性和精确性的重要性

---

## 核心概念

### 1. 好指令的五大标准

#### 标准 1：明确性（Clarity）

**定义**：指令清楚明白，不存在歧义

**评分标准**：
- ⭐ 差：读完后还不知道要做什么
- ⭐⭐ 一般：大概知道，但有多种理解方式
- ⭐⭐⭐ 好：读完后完全清楚要求
- ⭐⭐⭐⭐ 很好：即使第一次接触也能直接执行
- ⭐⭐⭐⭐⭐ 完美：极其清晰，无任何歧义

**评估问题**：
- [ ] 目标清晰吗？
- [ ] 成功的定义明确吗？
- [ ] 是否有歧义的地方？
- [ ] 外行能理解吗？

**例子对比**：

❌ **差的示例**：
```
"写一个 API"
```
问题：
- 什么类型的 API？
- 用什么框架？
- 什么功能？
- 返回什么？

✅ **好的示例**：
```
"用 Python + Flask 写一个 REST API，功能如下：
- 端点：GET /users/<id>
- 作用：返回指定用户的信息
- 返回格式：JSON，包含 id, name, email, created_at
- 错误处理：如果用户不存在返回 404
- 示例响应：
  {
    \"id\": 1,
    \"name\": \"John\",
    \"email\": \"john@example.com\",
    \"created_at\": \"2024-01-01\"
  }"
```

---

#### 标准 2：具体性（Specificity）

**定义**：指令包含足够的细节，而不是笼统抽象

**评分标准**：
- ⭐ 差：过于宽泛，有无限种理解方式
- ⭐⭐ 一般：有一些细节，但还不够完整
- ⭐⭐⭐ 好：具体到 AI 能准确执行
- ⭐⭐⭐⭐ 很好：有明确的标准和例子
- ⭐⭐⭐⭐⭐ 完美：细节到位，无需额外澄清

**评估问题**：
- [ ] 有具体的数字或参数吗？
- [ ] 有例子吗？
- [ ] 边界情况说明了吗？
- [ ] 标准和要求明确吗？

**例子对比**：

❌ **差的示例**：
```
"这个文档太长了，帮我总结一下"
```
问题：
- 多长才叫"总结"？100 字还是 1000 字？
- 要保留什么关键点？
- 什么风格？学术还是通俗？

✅ **好的示例**：
```
"总结这个文档为 200-300 字的摘要，包含以下要点：
1. 主要观点（1-2 句）
2. 关键数据或事实（3-4 项）
3. 结论或建议（1-2 句）
风格：专业但易理解，适合管理层阅读"
```

---

#### 标准 3：可验证性（Verifiability）

**定义**：指令的结果能被清晰地验证为成功或失败

**评分标准**：
- ⭐ 差：无法判断结果是否满足要求
- ⭐⭐ 一般：有模糊的成功标准
- ⭐⭐⭐ 好：有明确的检查点
- ⭐⭐⭐⭐ 很好：有具体的验收标准
- ⭐⭐⭐⭐⭐ 完美：有自动化的验证方式

**评估问题**：
- [ ] 怎样才能判断任务完成了？
- [ ] 成功的指标是什么？
- [ ] 有具体的验收标准吗？
- [ ] 能否自动验证？

**例子对比**：

❌ **差的示例**：
```
"写一个好的 README"
```
问题：
- "好"是什么意思？
- 包含什么内容？
- 多长合适？

✅ **好的示例**：
```
"为这个项目写 README，必须包含：
1. 项目描述（2-3 段）
2. 快速开始指南（包括安装和基本用法）
3. API 文档（表格格式，列出所有公共方法）
4. 常见问题（至少 3 个）
5. 贡献指南

验收标准：
- [ ] 无拼写错误
- [ ] 所有代码示例都能运行
- [ ] 包含至少一个截图
- [ ] 字数 800-1200 字"
```

---

#### 标准 4：完整性（Completeness）

**定义**：指令包含完成任务所需的所有信息

**评分标准**：
- ⭐ 差：缺少大量必要信息，AI 需要猜测
- ⭐⭐ 一般：缺少一些背景信息
- ⭐⭐⭐ 好：大部分信息完整
- ⭐⭐⭐⭐ 很好：所有必要信息都有
- ⭐⭐⭐⭐⭐ 完美：甚至包含预见的可能问题

**评估问题**：
- [ ] 提供了背景信息吗？
- [ ] 说明了为什么要这样做吗？
- [ ] 列出了所有约束条件吗？
- [ ] 提供了参考资源吗？

**例子对比**：

❌ **差的示例**：
```
"帮我审查这个代码"
```
问题：
- 代码在哪里？
- 重点审查什么？
- 用什么编程语言？
- 项目的背景是什么？

✅ **好的示例**：
```
"请审查这个 Python 数据处理函数。背景：
- 这是一个数据清洗管道的一部分
- 处理 100 万行的 CSV 数据
- 需要处理缺失值和异常值
- 性能很重要（必须 < 5 分钟）

请重点关注：
1. 算法效率（是否有优化空间？）
2. 错误处理（边界情况是否都处理了？）
3. 代码可读性

这是代码：
[代码...]"
```

---

#### 标准 5：可重复性（Reproducibility）

**定义**：相同的指令每次都能产生相似的结果，且结果可被他人复现

**评分标准**：
- ⭐ 差：结果不一致，难以复现
- ⭐⭐ 一般：大部分情况相同，但有时差异大
- ⭐⭐⭐ 好：通常能产生相似结果
- ⭐⭐⭐⭐ 很好：结果一致，易被他人复现
- ⭐⭐⭐⭐⭐ 完美：结果精确可控，可完全复现

**评估问题**：
- [ ] 固定了随机性因素吗？
- [ ] 版本信息清楚吗？
- [ ] 条件和前提明确吗？
- [ ] 别人能用同样指令得到同样结果吗？

**例子对比**：

❌ **差的示例**：
```
"给我一些创意的产品名字"
```
问题：
- 运行 10 次会得到 10 个完全不同的答案
- 无法保证质量
- 难以选择

✅ **好的示例**：
```
"为一个云存储产品生成 5 个创意名字。

要求：
1. 长度 1-3 个单词
2. 易于发音和记忆
3. 相关性：暗示云、存储、速度或安全的含义
4. 独特性：不是现有知名产品的变种
5. 品牌潜力：适合作为 SaaS 产品名

请按照这个格式返回，每行一个：
[名字] - 解释为什么这个名字适合

重要：请返回正好 5 个选项，不多不少。"
```

### 2. 指令质量自评表

使用这个表格评估你的指令质量：

| 标准 | 问题 | 分数(1-5) | 改进建议 |
|------|------|---------|---------|
| 明确性 | 指令清楚明白吗？ | ☐ | |
| 具体性 | 细节足够具体吗？ | ☐ | |
| 完整性 | 信息足够完整吗？ | ☐ | |
| 可验证性 | 能清楚验证结果吗？ | ☐ | |
| 可重复性 | 能重复产生相似结果吗？ | ☐ | |
| **总分** | **平均分** | **☐** | |

**评分标准**：
- 3 分以下：指令需要重写
- 3.5-4 分：指令还可以，有改进空间
- 4-4.5 分：指令较好，可在细节上改进
- 4.5+ 分：指令很好，可直接使用

---

### 3. 常见指令问题速查表

| 问题 | 症状 | 解决方案 |
|------|------|--------|
| **太笼统** | AI 返回的结果偏差大 | 添加具体例子和约束 |
| **信息不足** | AI 反复问澄清问题 | 提供更多背景信息 |
| **有歧义** | AI 理解错了你的意思 | 用多个角度说明 |
| **标准不清** | 难以判断结果是否满足 | 定义明确的验收标准 |
| **结果不稳定** | 每次运行结果差异大 | 固定参数，增加具体指导 |
| **太复杂** | AI 遗漏了重要细节 | 分解为更小的任务 |

## 实战案例

### 案例：指令改进过程

**初版指令**（得分 1.5/5）：
```
"写一篇关于AI的文章"
```

**改进第1次**（得分 2.5/5）：
```
"写一篇关于 AI 的文章，2000 字左右"
```

**改进第2次**（得分 3.5/5）：
```
"写一篇关于 AI 在教育中应用的文章
- 目标读者：教育管理者
- 字数：2000-2500 字
- 包含：背景、优势、挑战、案例
- 风格：专业、平衡、有说服力"
```

**改进第3次**（得分 4.5/5）：
```
"写一篇关于 AI 在教育中应用的文章

目标：说服学校管理者采用 AI 辅助教学
读者：无技术背景的校长和教务主任
字数：2000-2500 字

结构：
1. 开篇：一个吸引人的教育现状问题（100 字）
2. 解决方案：AI 能如何帮助（500 字）
   - 个性化学习
   - 教师负担减轻
   - 学生成果改善
3. 实际案例：2-3 个真实的学校案例（600 字）
4. 常见顾虑和回应（400 字）
   - 成本问题
   - 数据安全
   - 教师角色变化
5. 实施建议：3-5 个具体步骤（300 字）
6. 结尾：号召行动（100 字）

风格要求：
- 避免过度技术语言
- 使用具体数据和统计
- 包含 3-4 个关键的感悟点
- 乐观但现实，不过度宣传

示例开篇风格：
'...当今教室里，每个学生的学习进度各不相同，但传统教学...'

返回格式：
完整的文章，使用 Markdown 格式，包含标题和小标题"
```

**改进第4次**（得分 4.8/5）：
```
[同上，再加上]

验收标准（我会用这些来判断文章质量）：
- [ ] 文章长度 2000-2500 字（正文）
- [ ] 包含至少 2 个有名有姓的学校案例（Google Scholar 或新闻来源可验证）
- [ ] 至少 3 个具体数据点或统计
- [ ] 没有技术术语（或有简单解释）
- [ ] 论点逻辑清晰，能自圆其说
- [ ] 适合在校长会议上作为参考
- [ ] 无语法错误和拼写错误

如果您不确定某个案例或数据的准确性，请用 [待验证] 标记。"
```

## 关键要点总结

1. **明确性是基础**：让 AI 清楚了解你的要求
2. **具体性很关键**：越具体，AI 越能准确执行
3. **完整性很重要**：提供充分背景，减少 AI 的猜测
4. **可验证性必须**：定义清晰的成功标准
5. **可重复性有价值**：好的指令应该能反复使用
6. **自评很有用**：不断改进指令质量
