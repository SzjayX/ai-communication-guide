# 课程 06：指令结构万能公式（角色、任务、背景、步骤、格式）

## 学习目标
- 掌握最有效的指令结构模板
- 学会如何用这个公式改进现有指令
- 理解每个部分的作用

---

## 核心概念

### 1. 万能公式：RTBSF

```
R - Role（角色）：你是谁？
T - Task（任务）：要做什么？
B - Background（背景）：背景是什么？
S - Steps（步骤）：怎么做？
F - Format（格式）：结果什么样？
```

**这个公式为什么有效**：
- 涵盖了指令的所有关键要素
- 系统性强，不易遗漏
- 灵活性高，适用各种场景
- 经过实战验证，效果显著

---

### 2. 详解 RTBSF 的每个部分

#### 部分 R：Role（角色）

**定义**：定义 AI 应该扮演什么角色

**作用**：
- 激活 AI 的相关知识和专长
- 影响 AI 的表达风格和深度
- 帮助 AI 理解上下文

**例子**：
```
❌ 不明确的：
"帮我解释这个算法"

✅ 明确的角色：
"你是一位有 10 年计算机科学教学经验的大学教授。"
"你是一位资深的系统架构师。"
"你是一位有经验的 Python 开发者。"
```

**角色选择的几个维度**：

| 维度 | 选项 | 影响 |
|------|------|------|
| 职位 | 工程师、架构师、顾问、教授 | 深度和专业性 |
| 经验 | 初级、中级、高级、10 年+ | 答案的复杂度 |
| 风格 | 严肃、幽默、友好、学术 | 表达风格 |
| 背景 | 创业者、大公司、学术界 | 参考视角 |

**更强大的角色定义**：
```
"你是一位有 15 年经验的云架构师，
曾在 Google 和 Amazon 工作过，
现在是一家初创 SaaS 公司的 CTO。
你既理解大公司的规范，
也了解初创的灵活性和成本意识。"
```

---

#### 部分 T：Task（任务）

**定义**：明确要 AI 完成什么任务

**作用**：
- 激活相应的能力
- 聚焦 AI 的注意力
- 定义成功的标准

**常见的任务类型**：
```
分析、设计、编写、审查、解释、比较、
预测、规划、策略、建议、问题排查、优化、翻译、总结
```

**明确的任务表述**：
```
❌ 模糊：
"帮我改进这个代码"

✅ 清晰：
"帮我优化这个 Python 函数的时间复杂度，
从 O(n²) 降低到 O(n log n)"

✅ 更清晰：
"分析这个 Python 函数的性能瓶颈，
并建议优化方案，
目标是把时间复杂度从 O(n²) 降到 O(n log n)"
```

---

#### 部分 B：Background（背景）

**定义**：提供完成任务所需的所有背景信息

**包含的内容**：
- 为什么要这样做？
- 当前的情况是什么？
- 之前尝试过什么？
- 有什么限制或约束？

**背景的五个维度**（回顾第 5 课）：
1. 历史背景：项目的来源和演进
2. 业务背景：商业目标和用户
3. 技术背景：现有技术栈
4. 团队背景：团队能力和时间约束
5. 约束条件：限制和要求

**背景的表述方式**：
```
❌ 不完整：
"我们需要优化性能"

✅ 完整的背景：
"背景：
- 我们的应用有 100 万日活用户
- 最近用户投诉页面加载慢（> 3 秒）
- 这导致了 15% 的跳出率增加
- 我们已经做过基础优化（图片压缩、CDN）
- 初步分析表明瓶颈在数据库查询（每个页面 50+ 个查询）
- 我们的技术栈是 Django + PostgreSQL + Redis"
```

---

#### 部分 S：Steps（步骤）

**定义**：明确的步骤指导，或者问题的结构

**作用**：
- 如果提供了步骤，AI 会按步骤执行
- 即使没有明确要求，也能让 AI 组织得更好
- 帮助处理复杂任务

**两种使用方式**：

**方式 1：规定步骤**（当你知道怎么做时）
```
"请按以下步骤完成：
1. 首先，分析当前数据库查询的性能
2. 其次，识别最慢的 5 个查询
3. 然后，为每个查询设计优化方案
4. 最后，估算每个优化方案的改进效果"
```

**方式 2：让 AI 决定步骤**（当你不确定时）
```
"请以结构化的方式分析：
- 首先阐述问题的本质
- 然后列出可能的解决方案
- 接着分析各方案的优缺点
- 最后给出推荐方案"
```

**步骤的最佳实践**：
- 步骤通常是 3-7 个（太少了太笼统，太多了太复杂）
- 步骤要有逻辑顺序
- 每步的要求要清晰
- 可以为关键步骤添加具体要求

---

#### 部分 F：Format（格式）

**定义**：输出的格式、结构和风格

**作用**：
- 让 AI 的输出直接可用
- 减少后处理的工作
- 确保一致性

**常见的格式要求**：

**格式 1：结构化输出**
```
"请用以下格式返回：
- 问题描述：[...]
- 根本原因：[...]
- 解决方案（3 个）：
  1. [方案 1]
  2. [方案 2]
  3. [方案 3]
- 推荐方案：[...]
- 实施步骤：[...]"
```

**格式 2：代码输出**
```
"请返回完整的、可运行的 Python 代码：
- 包含必要的 imports
- 函数有完整的类型提示
- 包含 docstring
- 包含错误处理
- 可以直接 copy-paste 运行"
```

**格式 3：表格输出**
```
"请用表格比较，包含以下列：
- 方案名称
- 优点
- 缺点
- 实施难度
- 预期效果"
```

**格式 4：文档输出**
```
"请用 Markdown 格式，包含：
- 一级标题（主题）
- 二级标题（小节）
- 代码块（用 ``` 围起来，标注语言）
- 列表和表格
- 清晰的段落结构"
```

**格式 5：字数或长度要求**
```
"请控制在 500-800 字"
"请给出简明版本，不超过 3 段"
"请详细讲解，至少 2000 字"
```

### 3. RTBSF 指令模板

基础模板：

```
【角色】
你是一位 [具体描述角色和经验]。

【任务】
你的任务是 [明确描述要做什么]。

【背景】
背景信息：
- [背景要点 1]
- [背景要点 2]
- [背景要点 3]
...

【步骤】
请按以下步骤完成：
1. [第一步]
2. [第二步]
3. [第三步]
...

【格式】
请按以下格式返回：
[具体的格式说明]
```

### 4. 不同场景的 RTBSF 示例

#### 场景 1：代码审查

```
【角色】
你是一位有 10 年经验的资深 Python 工程师，
曾在 Google 做过系统性能优化。

【任务】
审查这个 Python 函数的代码质量，
找出性能瓶颈、安全问题和可维护性问题。

【背景】
- 这是一个数据处理管道的核心函数
- 每天处理 1000 万条数据
- 当前的处理时间是 2 小时
- 团队希望优化到 30 分钟以内
- 函数处理的数据是用户行为日志

【步骤】
1. 分析算法复杂度，识别瓶颈
2. 检查是否有安全漏洞（如 SQL 注入、数据泄露）
3. 评估代码可读性和可维护性
4. 给出 3 个优化建议，按优先级排序
5. 估算每个优化的改进效果

【格式】
返回一份代码审查报告：
- 总体评分：[优/良/中/需改进]
- 关键问题（如有）：[列表]
- 性能分析：[具体的复杂度分析]
- 优化建议（优先级排序）：
  1. [建议 1] - 预期提升 XX%
  2. [建议 2] - 预期提升 XX%
  3. [建议 3] - 预期提升 XX%
- 安全建议：[如有]
- 代码示例：[关键改进的示例代码]
```

#### 场景 2：技术方案设计

```
【角色】
你是一位系统架构师，
有多家互联网公司的大规模系统设计经验。

【任务】
设计一个支持千万级用户的实时推送系统。

【背景】
- 应用有 1000 万日活用户
- 需要支持实时消息推送（延迟 < 1 秒）
- 每天需要推送 5 亿条消息
- 目前用的是传统 HTTP 轮询方案，延迟太高
- 团队有 10 个工程师，熟悉 Node.js 和 Go
- 预算 200 万，时间 3 个月

【步骤】
1. 分析需求和现有问题
2. 对比 WebSocket、gRPC、消息队列等方案
3. 设计推荐方案的架构（包括各个组件）
4. 分析扩展性、可靠性、成本
5. 给出实施计划和风险评估

【格式】
返回一份技术设计文档（Markdown）：
- 需求分析：[整理需求]
- 方案对比表：[对比各种方案]
- 推荐架构：[有架构图描述]
- 组件设计：[每个关键组件的设计]
- 容量规划：[支持 1000 万用户的规划]
- 可靠性保证：[如何保证 99.9% 可用性]
- 实施计划：[按阶段的实施路线图]
- 风险评估：[可能的风险和应对方案]
```

#### 场景 3：内容创意

```
【角色】
你是一位有 5 年经验的产品文案顾问，
擅长为 SaaS 产品写高转化率的营销文案。

【任务】
为我们的云协作产品写一个 landing page 的主文案。

【背景】
- 产品：企业协作工具（类似 Slack，但更简化）
- 目标用户：10-100 人的小型创业团队
- 主要价值：降低沟通成本，提高协作效率
- 竞争对手：Slack（太复杂，太贵），微信（功能不够）
- 目标：获得 20% 的点击率
- 我们的差异：国产、便宜（1/10 的价格）、简单易用

【步骤】
1. 分析目标用户的痛点
2. 转化为产品的独特卖点
3. 设计文案的故事框架
4. 创作打动人心的文案
5. 提供 A/B 测试的多个版本

【格式】
返回文案内容（Markdown）：
- 版本 1：情感共鸣路线
  - 主标题（25 字以内）
  - 副标题（50 字以内）
  - 正文（200-300 字）
  
- 版本 2：利益驱动路线
  - 主标题
  - 副标题
  - 正文

- 版本 3：紧迫感路线
  - 主标题
  - 副标题
  - 正文

- 分析：为什么这 3 个版本可能有效
```

## 关键要点总结

1. **RTBSF 是万能公式**：Role 激活知识，Task 定义目标，Background 提供背景，Steps 指导过程，Format 规定输出
2. **角色很关键**：好的角色定义能显著提升 AI 的表现
3. **背景不能少**：背景直接影响 AI 理解的准确性
4. **格式很重要**：明确的格式要求能让输出直接可用
5. **灵活组合**：不是所有指令都需要 5 个部分，灵活调整
6. **迭代改进**：第一版指令可能不完美，通过反馈迭代改进

## 实践活动

### 练习 1：指令重构
选一个之前写过的指令，用 RTBSF 框架重新组织它。

### 练习 2：模板应用
用 RTBSF 模板为一个新任务写一个完整指令。

### 练习 3：对比测试
用同一个任务的两个版本（一个 RTBSF，一个非结构化），
对比 AI 的输出质量。
