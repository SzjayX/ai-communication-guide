## 定位

系统化思考是**高级指令设计**的基础。很多人觉得 AI 理解能力弱，其实是因为他们的**问题本身就没想清楚**。学会用思维导图整理思路，才能写出高质量指令。

---

## 核心观点

**AI 的能力上限 = 你的思路清晰程度**

一个模糊的问题，给 AI 再多信息也没用。但一个结构清晰的问题，AI 可以高效地帮你解决。

---

## 什么是系统化思考？

系统化思考 = 在问 AI 之前，先问自己这些问题：

### 思路检查清单

```
1️⃣ 我想解决的核心问题是什么？
   ├─ 能用一句话说清楚吗？
   └─ 还是有多个子问题？

2️⃣ 问题的背景是什么？
   ├─ 为什么需要解决这个问题？
   ├─ 解决好了的收益是什么？
   └─ 不解决会怎样？

3️⃣ 关键的限制条件是什么？
   ├─ 时间限制？
   ├─ 成本限制？
   ├─ 技术限制？
   └─ 其他约束？

4️⃣ 有哪些不同的方案？
   ├─ 方案 A 的优缺点？
   ├─ 方案 B 的优缺点？
   └─ 我倾向哪个？为什么？

5️⃣ 成功的标准是什么？
   ├─ 能测量吗？
   ├─ 什么样是"好"？
   └─ 什么样是"最好"？
```

## 实战案例：从模糊到清晰

### ❌ 模糊的问题

```
帮我生成一个营销策略
```

**问题**：
- 什么产品？
- 目标用户是谁？
- 预算多少？
- 时间期限？
- 成功标准？

AI 会很困惑，给出通用答案。

### ✅ 系统化思考后的问题

```
🎯 核心问题：
为 SaaS 产品（Claude API 集成工具）在未来 3 个月内获得 1000 个付费用户

📊 背景信息：
- 目标用户：Python 开发者（B2B SaaS 公司的 CTO 或 Tech Lead）
- 当前状态：已有 100 个免费用户，转化率 5%
- 资源限制：3 人小队，预算 50k 人民币，主要投在内容和社群

🎯 约束条件：
- 时间：3 个月
- 质量：用户留存率需 > 60%（不追求量，要质量）
- 渠道：主要线上（无 BD 团队）

📈 可选方案：
1. 内容营销（技术博客 + 开源项目）
2. 社群运营（Discord/微信技术社群）
3. 合作伙伴（集成到其他 SaaS 工具）
4. 产品驱动（优化免费版体验，增加转化）

🏆 成功标准：
- 主指标：30 个付费用户（相对保守）
- 次指标：留存率 > 60%
- 学习指标：找到最有效的获客渠道

❓ 我现在最不确定的是：哪个渠道性价比最高？
```

**现在 AI 有充分信息，可以**：
- 评估不同方案的可行性
- 给出针对性的建议
- 甚至帮你制定详细的 30 天行动计划

## 用思维导图整理复杂问题

### 使用场景：复杂的多层次问题

**问题**：设计一个内部协作平台

**用思维导图组织思路**：

```
🎯 内部协作平台设计
│
├─ 用户研究
│  ├─ 谁会用？（工程师、产品、设计、运营）
│  ├─ 他们的痛点？（沟通不畅、信息孤岛、决策缓慢）
│  └─ 期望的结果？（快速决策、知识共享、透明度）
│
├─ 核心功能
│  ├─ 消息沟通
│  │  ├─ 1:1 私聊
│  │  ├─ 团队频道
│  │  └─ 消息搜索
│  ├─ 知识管理
│  │  ├─ 文档库
│  │  ├─ Wiki
│  │  └─ 标签系统
│  └─ 任务协作
│     ├─ 任务看板
│     ├─ 进度追踪
│     └─ 评论讨论
│
├─ 技术选型
│  ├─ 前端：React/Vue
│  ├─ 后端：Node/Python
│  └─ 数据库：PostgreSQL
│
├─ 实现阶段
│  ├─ Phase 1（2 周）：消息沟通 MVP
│  ├─ Phase 2（2 周）：知识管理基础
│  └─ Phase 3（2 周）：任务协作
│
└─ 成功指标
   ├─ 采用率：50%+ 员工日活
   ├─ 满意度：NPS > 30
   └─ 效率提升：减少 30% 邮件数量
```

**用这个思维导图**，你可以：
- 清楚地向 AI 表述需求
- 让 AI 帮你设计 MVP 方案
- 评估不同的优先级
- 制定实现计划

## 系统化思考的 5 个工具

### 工具 1：问题分解（5 Why）

```
问题：为什么用户流失率 30%？
Why 1: 因为产品稳定性不好
Why 2: 为什么稳定性不好？因为基础架构有 Bug
Why 3: 为什么有 Bug？因为测试覆盖率只有 40%
Why 4: 为什么测试覆盖率低？因为没有专职 QA
Why 5: 为什么没专职 QA？因为预算有限

→ 根本原因：预算投入结构不合理

→ 给 AI 的问题会变成：
"我们想提高产品稳定性，但预算有限。
有没有高效的、成本较低的测试策略？"
```

### 工具 2：影响力矩阵

```
         重要性
         高    │
             │ ⭐ 立即做
             │   （重要+紧急）
             │
             │ ⭐ 计划做
             │   （重要+不紧急）
             ├─────────────────
     紧急性  │ ⚠️ 快速处理  │ 📌 排期
             │   （紧急+不重） │   （不重+不急）
         低  └─────────────────
             低           高
```

用这个矩阵决定：
- 什么先问 AI（紧急+重要的事）
- 什么不用问 AI（不紧急+不重要的事）

### 工具 3：利益相关者分析

```
这个决定影响谁？

┌─ 开发团队：增加 2 周工作量
├─ 用户：获得更好的体验
├─ 产品：核心竞争力提升
├─ 运维：维护成本增加 20%
└─ 管理层：需要新的预算

→ 权衡这些利益，给 AI：
"我们想做 X，但有以下约束和利益相关者考虑...
有没有能平衡各方诉求的方案？"
```

### 工具 4：约束条件清单

```
做这件事的约束：

📅 时间：3 个月
💰 预算：50k RMB
👥 人力：3 人团队
🛠️ 技术：只能用现有技术栈
📊 质量：不能降低现有功能稳定性
🎯 风险：不能影响核心产品

→ 给 AI：
"在以上约束下，有哪些可行的方案？
哪个风险最低？"
```

### 工具 5：决策树

```
应该做新功能还是优化现有功能？

            新功能 vs 优化
           /              \
        新功能          优化现有
       /    \          /      \
    风险高   风险低   易实现   难实现
    ...      ...     ...      ...
```

## 如何把系统化思考转化为指令？

### 指令模板

```
【背景和上下文】
- 问题是什么？
- 为什么重要？
- 相关的约束和限制？

【核心需求】
- 最主要的需求是什么？
- 有没有次要需求？

【约束条件】
- 时间限制？
- 成本/资源限制？
- 质量标准？
- 其他约束？

【利益相关者】
- 谁会受影响？
- 他们的诉求是什么？
- 有没有冲突的需求？

【可选方案】
- 方案 A：...
- 方案 B：...
- 方案 C：...

【成功标准】
- 什么样算成功？
- 如何衡量？

【我最关心的问题】
- 帮我评估这些方案的可行性
- 考虑约束条件，哪个最现实？
- 有没有我没想到的角度？
```

## 快速总结

✅ **系统化思考的核心**：
在问 AI 之前，先问自己 5 个问题（核心问题、背景、限制、方案、成功标准）

✅ **5 个工具**：
问题分解（5 Why）、影响力矩阵、利益相关者分析、约束清单、决策树

✅ **好处**：
- 思路清晰（自己先想通了）
- AI 理解准确（信息充分）
- 输出质量高（针对性强）
- 节省时间（少迭代）

✅ **一句话**：
清晰的思考 = 高质量的指令 = 优秀的 AI 输出
