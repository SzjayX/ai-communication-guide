## 常见错误 TOP 10

### ❌ 错误 1：信息不足，要求 AI 猜测

**表现**：
```
写一个营销文案
```

**问题**：
- 什么产品？
- 什么受众？
- 什么风格？
- 什么长度？

AI 会猜，结果 90% 不符合你的期望。

**✅ 改正**：
```
角色：营销文案专家
任务：为 SaaS 产品写推特文案
背景：
- 产品：AI 代码生成工具
- 受众：Python 开发者
- 核心卖点：提高效率 3 倍

输出要求：
- 长度：280 字以内
- 风格：专业但有趣
- 包含 1 个数据或具体场景

参考示例：
✅ "从写代码到 debug，Claude 帮我省了 2 小时每天。直接上手，0 配置。"
```

**教训**：永远供应充足的背景信息。

---

### ❌ 错误 2：要求过于宽泛，没有具体指标

**表现**：
```
帮我想个创新的产品idea
```

**问题**：
- "创新"的标准是什么？
- 给出 1 个还是 10 个idea？
- 应该考虑什么因素？

AI 会给出 "AI 工具"、"区块链应用" 这样的通用答案。

**✅ 改正**：
```
我想找到一个小而美的产品创意，满足以下条件：

目标市场：中国开发者，100k-500k 人
问题类型：工作效率或学习相关
技术难度：3 人小队能实现的（3-6 个月）
商业模式：SaaS 订阅或一次性收费
成功标准：6 个月内能达到 100 个付费用户

请给我 5 个具体的创意，每个包含：
- 解决的问题（要真实且普遍）
- 目标用户具体特征
- 竞争对手分析
- 为什么我的团队适合做这个
```

**教训**：把"宽泛的要求"转化为"具体的指标"。

---

### ❌ 错误 3：过度依赖 AI，失去主见

**表现**：
```
AI 说可以，就做了
```

**问题**：
- AI 不了解你的真实情况
- AI 可能"胡说八道"听起来很对
- 你失去了决策权

**✅ 改正**：
```
让 AI 帮助决策，而不是替代决策

当 AI 给出建议时：
1. 问自己：这符合我的价值观吗？
2. 挑战 AI：为什么这个方案比另一个好？
3. 验证：这个建议有数据支持吗？
4. 决定：我是否认同？
```

**教训**：AI 是工具，你是决策者。

---

### ❌ 错误 4：假设 AI 能记住上下文

**表现**：
```
第 1 条消息：我是一个 Python 开发者，想优化代码
第 2 条消息：(新对话) 帮我优化这个函数
```

**问题**：
- 第 2 条新对话中，AI 不知道你是 Python 开发者
- AI 可能给出 JavaScript 的建议

**✅ 改正**：
```
每个新的对话，重新提供关键背景：

"我是 Python 开发者，想优化代码效率。
这是我的函数：...
帮我优化，考虑到我的代码库用了 FastAPI 和 Pandas。"
```

**教训**：不要假设 AI 记得前面的内容。重要信息要重复。

---

### ❌ 错误 5：指令冲突，AI 不知道怎么办

**表现**：
```
指令 A：写得有趣一点，吸引眼球
指令 B：保持专业严肃的风格
```

**问题**：
- 这两个要求矛盾
- AI 会随机选一个，或者两个都做得不好

**✅ 改正**：
```
明确指令的优先级或界限：

"风格要求（按优先级）：
1. 首要：内容准确、专业
2. 其次：有一点幽默感（但不显得不专业）
3. 不要：浮夸、不严肃

具体说就是：
可以用数据 + 简短的例子让内容更有趣
但不要用表情符号、段子或过度的拟人化"
```

**教训**：避免相互矛盾的要求。如果有，要明确优先级。

---

### ❌ 错误 6：要求 AI 生成从未见过的格式

**表现**：
```
写一个 "蛇形叠层式的故事结构"
```

**问题**：
- 这个格式可能不存在或很罕见
- AI 可能"幻想"一个，听起来合理但完全是编的

**✅ 改正**：
```
要么用已知的格式名称：
"用英雄之旅的结构（Hero's Journey）讲这个故事"

要么详细描述你想要的结构：
"我想要这样的故事结构：
- 第 1 部分：问题出现（2 页）
- 第 2 部分：尝试解决（5 页）
- 第 3 部分：意外转折（3 页）
- 第 4 部分：最终解决（2 页）"
```

**教训**：要么用有名有姓的概念，要么明确描述。

---

### ❌ 错误 7：没有给出对错的标准

**表现**：
```
生成 10 个产品创意

AI 生成后：
"这些都不太对..."
```

**问题**：
- 你没有提前说什么样算"对"
- AI 在瞎猜你的期望

**✅ 改正**：
```
"生成 10 个产品创意，评估标准：
✅ 对的创意应该：
- 解决真实存在的问题（能举出具体场景）
- 目标市场 > 100k 用户
- 技术实现难度低于 6 个月
- 有清晰的商业模式

❌ 避免的创意：
- 太通用的想法（又一个 todo app）
- 技术难度过高（需要芯片设计）
- 市场太小（只有 1000 人）
- 法律风险（涉及个人隐私）"
```

**教训**：提前定义成功标准。

---

### ❌ 错误 8：忽视上下文长度限制

**表现**：
```
把整个 10 万字文档贴给 AI，问"总结一下"
```

**问题**：
- 可能超过 AI 的上下文窗口
- 即使不超，AI 也可能遗漏重要部分
- 效率低（AI 需要很长时间处理）

**✅ 改正**：
```
方法 1：分段处理
"这个文档分成 3 部分。先总结第 1 部分..."

方法 2：提前摘要
"这个 10 万字文档的核心是：[你的摘要]
基于这个背景，帮我..."

方法 3：问具体的部分
"这个文档第 5 章讲的是什么？"
```

**教训**：尊重 AI 的上下文限制。大任务分解成小任务。

---

### ❌ 错误 9：期望 AI 理解你的隐含需求

**表现**：
```
"帮我改进这个文案"

（没有说改进的方向）
```

**问题**：
- "改进"可能指：更短？更吸引？更专业？更有趣？
- AI 会随便改一些东西，可能不是你想要的

**✅ 改正**：
```
"这个文案要改进的方面（排优先级）：
1. 字数减少到 50 字以内
2. 加入一个数据或具体场景
3. 语气从'产品介绍'改为'用户故事'

当前版本：[粘贴文案]
参考目标：[给一个相似的好例子]"
```

**教训**：不要指望 AI 读心。隐含的需求要明确说出来。

---

### ❌ 错误 10：无限迭代，永远不满意

**表现**：
```
第 1 轮：不太对
第 2 轮：还是不太对
第 3 轮：还是有问题
...
第 10 轮：还是没完美
```

**问题**：
- 你的标准太完美主义
- 浪费时间和 token
- 最后的结果和第 5 轮可能差不多

**✅ 改正**：
```
定义一个"满意度标准"：

在开始迭代前，说清楚：
"成功标准是什么？满足这些条件就算完成。"

比如：
✅ 完成就是：
- 包含了 3 个主要卖点
- 长度 100-150 字
- 至少有 2 个用户能看到并同意

然后：迭代 3 轮，达到标准就停止。
```

**教训**：知道何时停止，完美是拖延症的面具。

## 防坑清单

使用前检查：

- [ ] 我提供了充足的背景信息吗？
- [ ] 我的要求有相互矛盾的地方吗？
- [ ] 我清楚地说明了成功标准吗？
- [ ] 我需要 AI 记住前面的信息吗？（如果是，要重复）
- [ ] 我要求的格式或概念 AI 能理解吗？
- [ ] 我的要求太宽泛了吗？（如果是，要加具体指标）
- [ ] 我设定了合理的期望吗？（完美≠追求）
- [ ] 我给出了示例（正例/反例）吗？
- [ ] 我明确了优先级吗？（如果有冲突需求）
- [ ] 我知道什么时候停止迭代吗？

## 快速总结

✅ **10 个常见错误总结**：

1. 信息不足 → 提供充分背景
2. 要求过于宽泛 → 加具体指标
3. 过度依赖 → 保持主见
4. 假设 AI 记住 → 重要信息重复
5. 指令冲突 → 明确优先级
6. 未知格式 → 用已知概念或详细描述
7. 没标准 → 提前定义成功标准
8. 忽视长度限制 → 分解大任务
9. 隐含需求 → 明确说出来
10. 无限迭代 → 定义满意度标准

✅ **一句话**：
好指令 = 充分信息 + 清晰需求 + 明确标准 + 实际期望
