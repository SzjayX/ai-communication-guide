## 技巧 1️⃣：思维链提示（Chain-of-Thought Prompting）

### 什么是思维链？

让 AI **展示推理过程**，而不是直接给答案。结果往往更准确。

### 例子：数学问题

❌ **直接问答**：
```
小明有 5 个苹果，妈妈给了他 3 个。然后他吃了 2 个。
现在他有几个苹果？
```

答案可能直接是数字，但推理过程不可见。

✅ **用思维链**：
```
小明有 5 个苹果，妈妈给了他 3 个。然后他吃了 2 个。
现在他有几个苹果？

请一步步思考：
1. 最初有多少？
2. 妈妈给了多少，现在总共有多少？
3. 吃了 2 个后，还剩多少？
4. 最终答案是多少？
```

输出会是：
```
1. 最初有 5 个苹果
2. 妈妈给了 3 个，所以 5 + 3 = 8 个
3. 吃了 2 个，所以 8 - 2 = 6 个
4. 最终答案是 6 个苹果
```

**优点**：
- 推理过程可验证
- 错误容易被发现
- 对复杂问题特别有效

### 例子：代码审查

❌ **直接问**：
```
这个代码有没有 Bug？
```

✅ **用思维链**：
```
这个代码有没有 Bug？

请按以下步骤分析：
1. 这段代码的目的是什么？
2. 输入可能是什么？输出应该是什么？
3. 逐行检查：每一行都做了什么？
4. 有没有边界情况没考虑到？
5. 有没有类型错误、空值错误或其他常见问题？
6. 总结：有几个 Bug，分别是什么？

代码：
[粘贴代码]
```

### 思维链的应用场景

✅ 适合：
- 逻辑推理（数学、编程、分析）
- 复杂决策
- 需要验证步骤的任务

❌ 不适合：
- 简单的信息查询（"2025 年是闰年吗？"）
- 需要立即答案的实时任务

## 技巧 2️⃣：角色扮演（Role-Playing / Persona）

### 什么是角色扮演？

让 AI **扮演一个特定身份**，这样能调整它的知识背景、说话风格、思维方式。

### 例子：产品评测

❌ **通用风格**：
```
评测这个产品
```

AI 给出标准的评测框架，但可能很无趣。

✅ **用角色扮演**：
```
你现在是一个发烧级的音乐爱好者，有 15 年的音频设备使用经验。
你对音质的要求极高，经常去音乐论坛讨论。
用这个身份评测这个蓝牙音箱，特别关注：
- 音质细节（高音、中音、低音的平衡）
- 与你用过的高端设备的对比
- 性价比（而不是绝对价格）

产品信息：[粘贴]
```

AI 的输出会更专业、更有深度，因为它在角色设定下思考。

### 角色设定的要素

```
你是：[具体身份]
背景：[多年经验/特殊背景]
风格：[说话方式、专业术语使用]
偏好：[关注的方面]
受众：[面对的听众]
```

### 例子：技术文档

```
你是一个资深的技术博主，专注于帮助初级开发者理解复杂概念。
你的特点：
- 用类比和现实例子，避免纯术语
- 经常问"这个有疑问吗？"来检查理解
- 会提到"新手常犯的错误"

现在，为一个初级开发者解释"Git 分支合并冲突"
```

### 角色扮演的应用场景

✅ 适合：
- 需要特定风格或专业背景的内容
- 需要特定受众视角的说明
- 想要模拟某个角色的回答

❌ 不适合：
- 需要纯粹客观信息的任务
- 需要多个角度平衡的分析

## 技巧 3️⃣：约束驱动（Constraint-Based Prompting）

### 什么是约束驱动？

用**明确的约束条件**来指导 AI 的行为，比给"建议"更有效。

### 例子：内容创作

❌ **模糊的约束**：
```
写一篇有趣的文章
```

✅ **清晰的约束**：
```
写一篇技术文章，满足以下约束：

字数约束：
- 总长 2000 字，不超过 2500 字
- 段落长度 80-150 字（便于阅读）

内容约束：
- 必须包含 3 个代码示例（Python）
- 每个示例 < 20 行
- 要有实际可运行的输出

风格约束：
- 禁用词：必然、绝对、永远
- 必用元素：1 个类比，1 个反例，1 个真实场景
- 语气：专业但亲切（像和朋友讲话）

结构约束：
- 开头：问一个问题（引起兴趣）
- 正文：4-5 个小节
- 结尾：总结 + 下一步学习建议

审核清单：
- [ ] 代码能直接复制运行吗？
- [ ] 有没有"必然""绝对"这类过度承诺的词？
- [ ] 新手能看懂吗？
```

现在 AI 有了明确的边界，输出会更符合预期。

### 约束的类型

```
📏 尺寸约束：
- 字数、行数、时间长度

🚫 禁止约束：
- 不要用某些词、不要某种风格

✅ 必须约束：
- 必须包含某些元素

🎯 风格约束：
- 语气、专业度、复杂度

📐 结构约束：
- 分几个部分，每部分多长

🔢 定量约束：
- 例子数量、数据格式等
```

### 约束驱动的威力

**例子：生成代码评论**

❌ **没有约束**：
```python
# Add validation
```
太笼统。

✅ **用约束**：
```
代码评论的约束：

风格约束：
- 一句话，不超过 60 字
- 说明"为什么"，不只说"做什么"
- 用现在时，不用命令式

内容约束：
- 如果涉及安全问题，要标注 [SECURITY]
- 如果涉及性能问题，要标注 [PERF]
- 如果是建议而非要求，用 "考虑" 开头

示例好评论：
// [SECURITY] 这里直接使用用户输入，容易被 SQL 注入。考虑使用参数化查询

示例不好评论：
// 不安全
```

输出会更一致、更专业。

## 三大技巧的组合使用

### 场景：评测一个新工具

```
你是一个资深的开发工具评测人（角色扮演），
用一步步的逻辑分析（思维链）来评测这个工具。

评测框架（约束驱动）：
1. 问题定义
   - 这个工具解决什么问题？
   - 为什么这个问题值得解决？

2. 功能评测
   - 核心功能是什么？
   - 相比竞品的优劣？

3. 使用体验
   - 上手难度？
   - 最流畅的使用场景？
   - 最卡的地方？

4. 价值评估
   - 对谁有价值？
   - 是否值得付费？

输出约束：
- 总长 1500 字
- 包含 1 个优点列表、1 个缺点列表
- 最后给出明确的建议（推荐/不推荐/有条件推荐）

工具信息：[粘贴]
```

## 何时用哪个技巧

| 场景 | 技巧 | 为什么 |
|------|------|--------|
| 需要严谨推理 | 思维链 | 显示过程，易验证 |
| 需要特定风格 | 角色扮演 | 调整 AI 的"身份" |
| 需要精确控制 | 约束驱动 | 明确边界 |
| 复杂任务 | 三个都用 | 最强效果 |

---

## 快速总结

✅ **三大技巧**：

1. **思维链** — 让 AI 展示推理步骤，提高准确率
2. **角色扮演** — 给 AI 一个身份，调整知识和风格
3. **约束驱动** — 用明确约束替代模糊建议，提高一致性

✅ **什么时候用**：
- 逻辑复杂 → 思维链
- 需要特定风格 → 角色扮演
- 要求精确 → 约束驱动
- 重要任务 → 组合使用

✅ **一句话**：
从"告诉 AI 做什么"升级到"用结构化的方式引导 AI"
